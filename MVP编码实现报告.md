## 3. 关键设计与决策

### 3.1 数据处理
- 行程子实体（天/景点/餐饮/住宿）通过 `raw_data` JSON 字段存储
- `to_dict()` 将 JSON 展开为 `itinerary_days`、`summary`、`tips` 键
- 模型字段（id/destination/days/budget）不会被 JSON 覆盖

### 3.2 容错机制
- **LLM 不可用** → 自动使用 Mock 数据，无需配置 API Key
- **天气不可用** → 跳过天气徽章，不影响主行程展示
- **数据损坏** → 返回空行程，不导致 500 错误
- **分享令牌无效** → 显示 404 错误页面

### 3.3 安全措施
- XSS 防护：JSON 数据通过 `type="application/json"` 脚本标签安全传递
- 分享令牌：UUID v4 随机生成，不可遍历猜测
- API Key：通过环境变量配置，.env 文件加入 .gitignore
- Session：配置 SECRET_KEY 加密

## 5. 已知问题与后续迭代建议

### 5.1 当前已知问题
1. **无 CSRF 保护**：POST 表单无 CSRF 令牌，MVP 阶段可接受
2. **无分页**：历史记录加载全部数据，适用于课程项目场景
3. **debug 模式默认开启**：部署到生产环境需关闭
4. **无用户认证**：所有用户匿名使用，无个性化体验

### 5.2 后续迭代建议
1. 集成 Flask-WTF 添加 CSRF 保护和表单验证
2. 添加 AJAX 提交和前端错误处理
3. 实现分页查询
4. 添加用户注册登录系统
5. 行程编辑/手动调整功能
6. 多目的地联程规划
7. 交通时间计算（调用地图 API）
