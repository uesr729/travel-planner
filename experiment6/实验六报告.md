### project-context Skill 内容

四阶段分析流程：
1. **项目级别识别** — 30 秒快速判断技术栈
2. **核心文件扫描** — 2 分钟扫描关键文件
3. **数据流追踪** — 1 分钟理清用户请求 → 路由 → 服务 → 数据库链路
4. **部署模式识别** — 单体/前后端分离/微服务

输出模板：包含项目名称、技术栈、架构模式、数据库、目录结构、核心文件角色表、数据流图描述、部署建议。

### coding-standards Skill 内容

六大规范：
1. **代码风格** — PEP 8、导入顺序、命名规范
2. **错误处理** — 分层处理 + 外部 API 降级策略 + 日志规范
3. **状态管理** — Loading/Empty/Error 三原则 + 表单处理
4. **安全实践** — API Key 管理、SQL 注入防护、XSS 防护
5. **文档规范** — 函数文档模板、模块文档要求
6. **Git 提交规范** — Conventional Commits

## 第二部分：Code Review

### 审查工具

- **ocr-review**：确定性缺陷扫描（NPE、SSRF、SQL注入、XSS）
- **code-review**：语义分析与架构审查（四阶段流程）

### ocr-review 扫描结果（7 项发现）

| # | 级别 | 类别 | 文件 | 问题 |
|---|------|------|------|------|
| 1 | 🔴 Blocking | 资源泄漏 | `llm_service.py:105` | OpenAI client 未做连接池管理 |
| 2 | 🔴 Blocking | NPE | `weather.py:46` | 字典直接下标访问（KeyError 风险） |
| 3 | 🟡 Important | 异常捕获过宽 | `weather.py:65` | `except Exception` 吞没所有异常 |
| 4 | 🟡 Important | 异常捕获过宽 | `mock_data.py:490` | 同上 |
| 5 | 🟡 Important | SSRF 风险 | `poi_service.py:84` | 用户输入拼接 URL 无防护 |

## 第三部分：Docker 容器化部署

### 容器架构

```
┌─────────────────────────────────────────────────────┐
│                   Docker Network                     │
│                  travel_network                      │
│                                                      │
│  ┌─────────────────┐   ┌─────────────────┐          │
│  │  前端容器        │   │  后端容器        │          │
│  │  travel-frontend │   │  travel-backend │          │
│  │  Nginx :80       │──▶│  Gunicorn :8000 │──┐       │
│  │  代理 / → backend│   │  Flask 应用     │  │       │
│  └─────────────────┘   └─────────────────┘  │       │
│             ↑                    ↑           │       │
│       用户浏览器           服务名 backend     │       │
│                                             │       │
│                                      ┌──────┘       │
│                                      ▼               │
│                              ┌─────────────────┐     │
│                              │  数据库容器       │     │
│                              │  travel-db       │     │
│                              │  PostgreSQL :5432 │     │
│                              │  持久化 volume    │     │
│                              └─────────────────┘     │
└─────────────────────────────────────────────────────┘
```

### 关键设计点

| 维度 | ✅ 正确做法 | ❌ 避免做法 |
|------|-----------|-----------|
| 数据库地址 | `postgresql://travel:travel123@**db**:5432/...` | `@localhost:5432` |
| 后端监听 | `--bind 0.0.0.0:8000` | `127.0.0.1` |
| Nginx 代理 | `proxy_pass http://**backend**:8000` | `localhost:8000` |
| 数据持久化 | `volumes: travel_db_data:/var/lib/postgresql/data` | 不配置 volumes |
| 启动顺序 | `depends_on: db: condition: service_healthy` | 无 healthcheck |

### 文件清单

```
experiment6/
├── backend/
│   └── Dockerfile              # 后端容器（Python 3.13 + Gunicorn）
├── frontend/
│   ├── Dockerfile              # 前端容器（Nginx 多阶段构建）
│   └── nginx.conf              # Nginx 反向代理配置
```

### 启动命令

```bash
# 构建并启动所有容器
docker compose -f experiment6/docker-compose.yml up -d

# 查看容器状态
docker compose -f experiment6/docker-compose.yml ps

# 查看日志
docker compose -f experiment6/docker-compose.yml logs -f

# 停止所有容器
docker compose -f experiment6/docker-compose.yml down
```
