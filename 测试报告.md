## 5. 典型 Bug 分析

### BUG-001：to_dict() 字段覆盖

**问题描述**：`Itinerary.to_dict()` 方法中，`**data`（raw_data 展开）放在返回字典的最后，导致如果 raw_data JSON 中包含 `id`、`days`、`destination` 等键，会覆盖模型本身的字段值。

**发现方式**：代码审查阶段，审查 Agent 指出 `**data` 展开在模型字段之后存在风险。

**根因分析**：
- `to_dict()` 的实现是：先放模型字段，然后 `**data` 展开
- raw_data 来自 LLM 或 Mock 生成，可能包含任意键
- 如果 LLM 输出意外包含 `"id": "malicious"`，`to_dict()` 返回的 `id` 将是字符串而非整数

**修复方案**：
```python
# 修改前（风险：raw_data 覆盖模型字段）
return {"id": self.id, ..., **data}

# 修改后（只提取预期键，避免覆盖）
return {
    "id": self.id, ...,  # 模型字段优先
    "itinerary_days": data.get("days", []),
    "summary": data.get("summary", {}),
    "tips": data.get("tips", []),
}
```

**影响范围**：影响所有调用 `to_dict()` 的视图函数（view_itinerary、share_itinerary），以及对应的 Jinja2 模板中对 `data.days` 的引用（需改为 `data.itinerary_days`）。
