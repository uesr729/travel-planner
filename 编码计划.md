## 4. 关键技术决策

### 4.1 数据存储策略
- 行程子实体（天/景点/餐饮/住宿）作为 JSON 嵌入 `raw_data` Text 字段
- **理由**：写一次读多次模式，JSON 存储减少表连接，页面渲染更快
- **权衡**：无法对子实体做 SQL 查询，MVP 阶段不需要

### 4.2 API 容错策略
- LLM API 超时 30 秒，失败自动切换 Mock 数据
- 天气 API 超时 10 秒，失败自动切换 Mock 数据
- 切换过程对路由层透明，不通知用户

### 4.3 前端渲染策略
- 服务端渲染（SSR）通过 Jinja2 模板
- 地图使用 Leaflet.js + OpenStreetMap 免费图层
- 偏好标签通过点击式标签选择器实现
