# 软件工程实验项目文档

## AI 旅行规划系统

---

# 第1章 绪论

## 1.1 项目背景

随着人们生活水平的提高和出行需求的增长，旅行规划成为旅游消费中不可或缺的一环。然而，传统的旅行规划方式存在以下痛点：

- **信息碎片化**：用户需要在多个平台分别查找景点、餐饮、住宿、天气等信息，效率低下
- **规划耗时长**：一次完整的行程规划平均需要 2-3 小时
- **个性化不足**：通用攻略无法根据个人偏好（如美食、文化、自然）定制行程
- **费用难估算**：缺乏整合的费用预估，实际消费常超出预算

AI 大模型的发展为解决这些问题提供了新思路。通过自然语言生成能力，AI 可以根据用户的目的地、天数、预算和偏好，一键生成完整的行程规划。

## 1.2 研究现状

目前市面上的旅行规划工具主要分为三类：

| 类型 | 代表产品 | 优点 | 缺点 |
|------|---------|------|------|
| 传统攻略平台 | 马蜂窝、穷游 | 内容丰富 | 搜索筛选成本高 |
| 在线旅游平台 | 携程、飞猪 | 预订一体化 | 行程规划功能弱 |
| AI 辅助工具 | ChatGPT、文心一言 | 生成速度快 | 缺乏结构化输出 |

现有方案的主要不足在于：旅行社群平台依赖 UGC 内容，质量参差不齐；通用 AI 工具虽能生成行程但格式不统一，难以直接落地使用。

## 1.3 项目目标

本项目旨在开发一个 AI 驱动的智能旅行规划系统，实现以下目标：

1. **一键生成**：用户输入目的地、天数、预算和偏好，系统自动生成完整行程
2. **真实数据**：集成高德地图 API，获取真实景点 POI 数据
3. **可视化展示**：通过 Leaflet 地图标注每日景点，清晰展示路线
4. **费用管理**：自动计算景点、餐饮、住宿的各项费用
5. **分享功能**：一键生成分享链接，方便与家人朋友共享

# 第2章 相关技术

## 2.1 后端技术栈

### 2.1.1 Python Flask 框架

Flask 是一个轻量级的 Python Web 框架，具有以下优势：
- **灵活性高**：采用微内核架构，按需扩展功能
- **学习曲线平缓**：API 简洁，适合快速开发
- **生态丰富**：Flask-SQLAlchemy、Flask-Login 等扩展覆盖常见需求
- **模板引擎**：内置 Jinja2 模板引擎，支持服务端渲染

### 2.1.2 Flask-SQLAlchemy ORM

SQLAlchemy 是 Python 最成熟的 ORM 框架，与 Flask 配合使用：
- 无需手写 SQL，通过 Python 对象操作数据库
- 支持 SQLite 和 PostgreSQL 切换
- 提供数据库迁移能力（Flask-Migrate 扩展）

### 2.1.3 Gunicorn 生产部署

Gunicorn 是 Python WSGI HTTP 服务器，用于生产环境部署：
- 多 worker 进程并发处理请求
- 支持热重载
- 与 Nginx 配合构建完整的生产架构

## 2.2 前端技术栈

### 2.2.1 Jinja2 模板引擎

Jinja2 是 Flask 默认的模板引擎，支持模板继承、宏、过滤器等特性，适合服务端渲染场景。

### 2.2.2 Leaflet.js 地图库

Leaflet 是一个轻量级开源地图库，主要特点：
- 无需 API Key，使用 OpenStreetMap 免费瓦片
- 支持标记点、路线绘制、弹窗交互
- 移动端适配良好
- 插件生态丰富

## 2.3 外部 API

### 2.3.1 高德地图 Web 服务 API

高德地图提供 POI（兴趣点）搜索 API，可通过 HTTP 接口查询任意城市的景点、餐饮、购物等信息。本项目将其作为真实景点数据源。

### 2.3.2 DeepSeek API（可选）

DeepSeek 提供与 OpenAI 兼容的 Chat API，可用于 AI 生成行程规划。未配置 API Key 时自动降级为 Mock 数据。

### 2.3.3 OpenWeather API（可选）

OpenWeather 提供 5 天天气预报 API，用于展示目的地的天气信息。未配置时自动降级为 Mock 数据。

## 2.4 部署技术

### 2.4.1 Docker 容器化

Docker 提供轻量级容器化方案，实现环境一致性和快速部署：
- 镜像构建 → 容器运行 → 服务编排
- docker-compose 支持多容器编排（前端 + 后端 + 数据库）

### 2.4.2 Railway 云部署

Railway 是 PaaS 云平台，支持从 GitHub 仓库自动构建和部署 Python 应用，提供公网域名和 HTTPS。

## 3.3 业务流程

用户使用系统的完整流程如下：

1. 用户访问首页，填写表单（目的地、天数、预算、偏好）
2. 前端校验必填字段（目的地不能为空）
3. 提交 POST 请求到后端 `/generate` 路由
4. 后端调用 LLM 服务（或 Mock 数据）生成行程 JSON
5. 同时获取目的地天气数据
6. 保存行程到数据库（含分享 token）
7. 重定向到行程详情页
8. 前端加载 Leaflet 地图，标注景点位置
9. 用户可打印/分享行程

首页包含行程生成表单，用户输入目的地、选择天数和预算、添加偏好标签。表单提交时显示加载动画。

```python
# routes/main.py - 首页路由
@main_bp.route("/")
def index():
    return render_template("index.html")

# 表单提交处理
@main_bp.route("/generate", methods=["POST"])
def generate():
    destination = request.form.get("destination", "").strip()
    days = request.form.get("days", 3, type=int)
    budget = request.form.get("budget", 3000, type=int)
    preferences = request.form.get("preferences", "").strip()
    
    raw_data = generate_itinerary(destination, days, budget, preferences)
    itinerary = Itinerary(destination=destination, days=days,
                          budget=budget, preferences=preferences,
                          raw_data=raw_data)
    db.session.add(itinerary)
    db.session.commit()
    return redirect(url_for("main.view_itinerary", id=itinerary.id))
```

## 5.2 行程生成引擎

系统采用三层数据优先级策略：

1. **真实 POI 数据**：配置高德地图 API Key 时，获取真实景点数据
2. **核心城市预设**：北京、上海、成都、西安、杭州 5 城有精心准备的景点
3. **通用生成**：其他城市自动生成合理的通用行程

```python
# mock_data.py - 数据优先级实现
def _build_city_data_with_real_poi(city: str) -> dict:
    if city in CITY_DATA:
```

## 5.3 高德 POI 搜索服务

通过高德地图 Web 服务 API 搜索真实景点：

```python
# poi_service.py - 搜索景点
def search_attractions(city: str) -> list:
    api_key = current_app.config.get("AMAP_API_KEY", "")
    if not api_key:
        return []
    
    all_results = {}
    keywords = ["景点", "风景区", "公园", "博物馆"]
    
    for keyword in keywords:
        results = _amap_text_search(api_key, keyword, city)
        for poi in results:
            name = poi.get("name", "")
            if name and name not in all_results:
                all_results[name] = poi
    
    # 过滤无效坐标
    return [p for p in all_results.values()
            if abs(p.get("lat", 0)) > 0.001]
```

## 5.4 地图可视化

使用 Leaflet.js 在地图上标注所有景点，按天着色并绘制路线：

```javascript
// main.js - 地图初始化
function initMap(days) {
    var allSpots = [];
    days.forEach(function(day) {
        day.spots.forEach(function(spot) {
            allSpots.push({
                name: spot.name, lat: spot.lat,
                lng: spot.lng, day: day.day_number
            });
        });
    });
    
    // 计算地图边界
    var bounds = L.latLngBounds();
    allSpots.forEach(function(spot) {
        bounds.extend([spot.lat, spot.lng]);
    });
    
    var map = L.map('map').fitBounds(bounds, {padding: [40,40]});
    L.tileLayer('https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png')
        .addTo(map);
    
    // 添加标记和路线
    allSpots.forEach(function(spot) {
        L.circleMarker([spot.lat, spot.lng], {radius: 8})
            .addTo(map);
    });
}
```
